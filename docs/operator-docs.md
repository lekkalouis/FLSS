# FLSS Operator Runbooks

## Purpose
This guide is for dispatch, warehouse, and customer support operators using the FLSS console.

## Daily operating modules
- **Dispatch Console (`/scan`)**
  - Scan parcels, validate order matching, and initiate booking.
- **Order Operations Board (`/ops`)**
  - Triage open orders and work through packing/dispatch exceptions.
- **Fulfillment Timeline (`/fulfillment-history`)**
  - Review shipped, delivered, pickup-ready, and collected outcomes.
- **Customer Directory (`/contacts`)**
  - Search customer records, provinces, and account tiers.
- **Print Station (`/print-station`)**
  - Run template prints, monitor print history, and trigger reprints.
- **Process Blueprints (`/flowcharts`)**
  - Follow standard dispatch and packing process maps.
- **Knowledge Hub (`/docs`)**
  - Open role-specific docs directly from the app.

## Operational checklists
- Confirm integrations are healthy from status indicators before processing.
- Validate scan inputs before booking labels.
- Reconcile fulfillment outcomes at end-of-day.
- Escalate booking/fulfillment failures to admin with order numbers and timestamps.
